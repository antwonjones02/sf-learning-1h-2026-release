Multiple Employments REST API Enhancements

Feature #2 of 54

What Is This Feature?

The Multiple Employments REST API now provides structured responses, supports $select for retrieving specific fields, and introduces the actingAs parameter.

What Is Changing?

API responses are now more structured and efficient. Developers can request only the data fields they need and act on behalf of specific users.

Why Should We Care?

Enables more efficient integrations for organizations with employees who hold multiple roles. Reduces data transfer overhead.

Risk of Inaction

No action needed — automatically applied. Existing integrations continue to work.
